Approach

How I like to build.

I translate messy requirements into usable systems. That usually means combining code, spreadsheets, and documentation instead of pretending one tool solves everything. I value reproducibility, explicit assumptions, good naming, source-of-truth discipline, and outputs another person can understand without guessing.

Reproducible workflows Auditable logic Clean documentation Structured outputs Maintainable scripts Practical automation
